# LLM-Wiki 知识库系统 使用说明书

## 版本：v3.0 | 更新日期：2026-05-03

---

## 一、系统概述

LLM-Wiki 是一个基于 LLM 的知识编译系统，将学术论文自动转化为结构化、可追溯、可复用的 Wiki 知识库。

### 核心能力

- **PDF 自动转换**：将 PDF 论文转换为结构化 Markdown
- **智能内容生成**：一次 LLM 调用生成论文摘要 + 实体 + 概念
- **语义搜索**：基于 sentence-transformers 的向量检索 + BM25关键词检索混合搜索
- **程序化质量审核**：5维度评分 + LLM修正循环
- **跨论文知识融合**：LLM判断+论文上下文的智能融合
- **知识图谱可视化**：实体/概念关系图谱展示
- **Markdown编辑**：原文编辑/预览/分屏三模式，KaTeX公式渲染
- **PDF对比查看**：左侧Markdown+右侧PDF原文并排审核
- **系统健康体检**：两层检查（规则+LLM增强），一键自动修复

### 系统架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│  前端 (React) │ ←→ │ 后端 (FastAPI) │ ←→ │  知识库 (Wiki) │
│  端口: 5173   │     │  端口: 8000   │     │  Obsidian Vault│
└─────────────┘     └──────────────┘     └──────────────┘
                           ↕
                    ┌──────────────┐
                    │  LLM 服务     │
                    │  (本地/远程)   │
                    └──────────────┘
```

## 四、功能使用

### 4.1 论文导入

**入口**：侧边栏 → "论文导入"

1. 上传 PDF 文件或选择已有 PDF
2. 系统自动调用 marker-pdf 转换为 Markdown
3. 转换完成后，原文出现在"原文"列表中

### 4.2 论文生成

**入口**：侧边栏 → "论文生成" 或 API 批量处理

**单篇生成**（前端操作）：
1. 在"原文"列表中选择一篇文档
2. 点击"生成"按钮
3. 系统调用 Agent-G 生成论文 + 实体 + 概念
4. 生成完成后自动进入审核流程

**批量生成**（命令行）：
```bash
cd E:\ragtest
python -m scripts.batch --batch-size 5 --max-batches 1
```

参数说明：
- `--batch-size N`：每批处理 N 篇文档
- `--max-batches N`：最多处理 N 批
- `--reset`：重置处理状态，从头开始
- `--status`：查看当前处理进度
- `--dry-run`：仅扫描，不实际处理

### 4.3 生成流程详解

系统采用"先分析后生成"的三步策略：

```
第一步：结构分析
  → 识别论文类型（常规论文/综述/技术报告）
  → 列出章节和核心内容
  → 提取核心推理链
  → 确定必须覆盖的内容

第二步：内容生成
  → 论文：基本信息、核心观点、摘要、问题定义、方法、实验、结论
  → 实体：人名、机构、数据集等（中文-英文格式）
  → 概念：算法、方法论、技术思想等（中文-英文格式）

第三步：覆盖完整性自检
  → 逐项检查必须覆盖的内容
  → 评分并决定是否通过
```

### 4.4 程序审核（Agent-R）

审核采用程序规则打分（非 LLM 审核），5个维度：

| 维度 | 权重 | 检查内容 |
|------|------|----------|
| 完整性 | 25% | 关键章节是否存在、方法长度、实验数值 |
| 准确性 | 25% | Source ID 比例（论文满分，实体/概念需检查） |
| 结构规范性 | 20% | Frontmatter 格式、必填字段 |
| 可发现性 | 15% | 交叉引用数量、Wiki 链接 |
| 冲突处理 | 15% | Conflict 标记格式 |

**通过标准**：综合分 ≥ 7.5

**修正循环**：
- 审核不通过 → 调用 LLM 修正 → 重新审核
- 最多 3 轮修正
- 3 轮后仍不通过 → 标注"待完善"并存档

### 4.5 实体/概念融合

当新论文生成的实体/概念与知识库中已有的重复时，系统使用 LLM + 论文上下文判断是否融合：

**重复识别**（4层匹配）：
1. 英文标题精确匹配
2. 中文标题精确匹配
3. 英文归一化匹配
4. 模糊匹配（相似度 ≥ 0.85）

**融合决策**（LLM判断）：
- 提取两篇论文中实体/概念出现的上下文片段
- 将上下文交给LLM判断是否为同一实体/概念
- LLM决定：融合（给出合并方案）或不融合（给出理由并记录）

### 4.6 知识浏览

**论文详情页**：
- 显示论文基本信息、核心观点、摘要、方法、实验等
- 顶部展示提取的实体（绿色标签）和概念（橙色标签）
- 标签可点击跳转到对应页面

**实体/概念页面**：
- 显示定义、关键信息、引用来源、相关论文
- 引用来源标注 `[Source: 论文标题]`
- 相关论文以 Wiki 链接形式展示

### 4.7 原文编辑与审核

**入口**：知识库 → 选择原文 → 点击文件名

**编辑模式**：
- **编辑模式**：直接编辑 Markdown 源码
- **预览模式**：渲染后的效果预览（含 KaTeX 公式渲染）
- **分屏模式**：左侧编辑 + 右侧实时预览

**PDF对比查看**：
- 点击"PDF对比"按钮
- 左侧显示 Markdown 渲染结果
- 右侧显示 PDF 原文
- 便于审核转换质量

**文件重命名**：
- 点击文件名旁的编辑图标
- 修改文件名后保存

**KaTeX公式渲染**：
- `$...$` 行内公式和 `$$...$$` 块级公式自动渲染
- 支持矩阵、分数、积分等常见数学公式

### 4.8 知识图谱

**入口**：侧边栏 → "知识图谱"

展示实体/概念之间的关系网络，支持：
- 缩放、拖拽
- 节点点击跳转
- 布局切换

### 4.9 智能查询

**入口**：侧边栏 → "智能查询"

输入自然语言问题，系统检索知识库并生成回答。

**搜索机制**：
- 向量检索（权重70%）：使用 all-MiniLM-L6-v2 语义嵌入模型，支持语义相似度匹配
- 关键词检索（权重30%）：BM25算法，支持中英文混合分词
- 标题匹配增强：查询词与页面标题匹配时，概念/实体页排名提升
- 融合排序：加权融合向量检索和关键词检索结果

**搜索范围**：
- 原始文档（raw/papers/markdown/）
- 论文页面（wiki/papers/）
- 实体页面（wiki/entities/）
- 概念页面（wiki/concepts/）

### 4.10 系统健康体检

**入口**：侧边栏 → "系统状态"

**第一层：快速体检**（规则检查，5-10秒）：
- 孤儿页面检测（存在但未被引用的页面）
- 未解决矛盾标记检测
- Frontmatter 完整性检查
- 论文完整性检查（缺失章节）
- 交叉引用断裂检测（引用不存在的页面）
- 重复实体/概念检测
- 缺失概念页面检测
- 实体/概念无Source标注检测

**第二层：深度体检**（LLM增强，1-3分钟）：
- LLM概念建议（分析论文内容建议新概念）
- 质量抽检（随机抽样评估）

**体检结果展示**：
- 健康分数（0-100分）
- 各维度得分（孤儿页面、矛盾标记、Frontmatter、论文完整性等）
- 问题详情表格（含操作按钮）

**一键修复操作**：

| 问题类型 | 操作 | 说明 |
|---------|------|------|
| 无效Frontmatter | 自动修复 | 从内容中提取标题写入Frontmatter |
| 论文不完整 | 删除并重新生成 | 删除论文及关联实体/概念，从源文件重新生成（后台异步执行） |
| 交叉引用断裂 | 删除链接 | 移除页面中无法解析的链接 |
| 重复实体/概念 | 合并 | 选择保留项和删除项，合并内容 |

### 4.11 用户管理

**入口**：侧边栏 → "用户管理"（仅管理员可见）

功能：
- 查看用户列表（用户名、邮箱、角色、状态、注册时间）
- 修改用户角色（管理员/系统维护/核心用户/一般用户）
- 启用/禁用用户
- 重置用户密码（重置为默认密码 123456）

### 4.12 日志管理

**入口**：侧边栏 → "日志管理"（管理员和系统维护可见）

功能：
- 用户操作日志：登录、登出、查询、浏览实体/概念等操作记录
- 系统运行日志：系统级别的事件记录
- 关键词搜索和分页
- 时间显示为北京时间

### 4.13 暗色模式

**切换**：顶部导航栏右侧切换按钮

- 全局CSS变量驱动，所有页面元素适配
- Ant Design组件样式覆盖
- 知识图谱节点颜色适配

## 五、命令行工具

### 5.1 批量处理

```bash
# 处理所有待处理文档
python -m scripts.batch

# 每批5篇，最多处理2批
python -m scripts.batch --batch-size 5 --max-batches 2

# 查看当前进度
python -m scripts.batch --status

# 重置进度，从头开始
python -m scripts.batch --reset

# 仅扫描，不实际处理
python -m scripts.batch --dry-run
```

### 5.2 单篇生成

```bash
# 生成论文+实体+概念
python -m scripts.agent_g --input raw/papers/markdown/xxx.md

# 仅生成论文
python -m scripts.agent_g --input raw/papers/markdown/xxx.md --paper-only
```

### 5.3 审核

```bash
# 审核单个页面
python -m scripts.review --input wiki/papers/xxx_论文.md

# 审核所有未审核页面
python -m scripts.review --all
```

### 5.4 系统体检

```bash
# 运行完整体检
python -m scripts.lint

# 仅检查 Source ID 比例（阈值30%）
python -m scripts.lint --source-ratio 0.3
```

体检项目：
1. 孤儿页面检测
2. 未解决矛盾标记检测
3. Frontmatter 完整性检查
4. 论文完整性检查
5. 交叉引用断裂检测
6. 重复实体/概念检测
7. 缺失概念页面检测
8. Source ID 比例检测（仅实体/概念）

### 5.5 PDF 转换

```bash
# 转换单个 PDF
python -m scripts.pdf_converter --input raw/papers/pdf/xxx.pdf

# 批量转换
python -m scripts.pdf_converter --input raw/papers/pdf/ --batch
```

## 八、常见问题

### Q1：PDF 转换失败怎么办？

系统优先使用 marker-pdf，失败后自动回退到 PyMuPDF。如果两者都失败：
1. 检查 PDF 文件是否损坏
2. 确认 marker-pdf 已安装：`pip install marker-pdf`
3. 确认 PyMuPDF 已安装：`pip install PyMuPDF`

### Q2：生成内容不完整怎么办？

增强版 Prompt 采用"先分析后生成"策略，通常会覆盖全文。如果仍有遗漏：
1. 检查原始 Markdown 是否完整
2. 检查 LLM 服务的 max_tokens 配置（建议 ≥ 4000）
3. 查看审核评分中的具体扣分项
4. 使用系统健康体检中的"删除并重新生成"功能

### Q3：实体/概念重复了怎么办？

系统自动通过4层匹配+LLM判断识别重复并融合。如果仍有重复：
1. 在系统健康体检中查看"重复实体/概念"
2. 点击"合并"按钮，选择保留项和删除项
3. 系统自动合并内容并更新引用

### Q4：图片无法显示怎么办？

1. 确认后端静态文件服务已启动
2. 检查图片路径是否在 `raw/papers/markdown/` 目录下
3. 确认 `/api/assets` 端点可访问

### Q5：审核评分太低怎么办？

查看审核报告中的具体扣分项：
- 完整性不足：补充缺失章节
- 准确性不足（实体/概念）：添加 Source ID 标注
- 结构不规范：修正 Frontmatter 格式
- 可发现性不足：增加交叉引用链接

### Q6：暗色模式显示异常怎么办？

系统已全面适配暗色模式。如果发现个别元素显示异常：
1. 刷新页面
2. 切换回亮色模式再切换回来
3. 清除浏览器缓存

### Q7：论文重新生成需要多长时间？

论文重新生成是异步操作，通常需要3-10分钟（取决于LLM服务响应速度）。操作后：
1. 系统立即删除旧论文及关联页面
2. 后台启动重新生成流程
3. 前端每5秒轮询任务状态
4. 完成后自动显示成功消息

### Q8：原文编辑后保存失败怎么办？

1. 确认后端服务正在运行
2. 检查浏览器控制台是否有CORS错误
3. 确认文件路径中没有特殊字符
