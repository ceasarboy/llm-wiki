## 会议讨论总结

### 1. 核心问题确认

**问题**：论文生成时可能不覆盖文章全部内容，特别是方法论推理的逻辑链可能断裂。

### 2. 关键洞察

1. **内容截断影响**：
   - 当前方案：只取前25段 × 1000字 = 25000字
   - 评估：对于论文来说接近上下文最大容量，问题不大
   - 真正问题：**段落间的逻辑联系可能被切断**

2. **结构多样性问题**：
   - 常规论文：有标准IMRaD结构
   - 综述文章：按主题组织，没有强推理链
   - 结论：需要通用性更强的方案

3. **LLM能力确认**：
   - Qwen3.5 27B：262K context，65K输出
   - DeepSeek-V4 Flash：1M context，384K输出
   - **结论**：Context Window完全足够

### 第二步：双Agent架构测试（后续任务）

通过第一步验证后，测试两种审核方案：

#### 方案A：两次独立调用

```
调用1：Agent-G 生成
┌─────────────────────────────────────┐
│ 输入：论文 + 生成指令                 │
│ 输出：生成的Wiki内容                  │
└─────────────────────────────────────┘

调用2：Agent-R 审核
┌─────────────────────────────────────┐
│ 输入：生成内容 + 论文 + 审核指令      │
│ 输出：审核结果 + 评分                 │
└─────────────────────────────────────┘
```

**优势**：
- ✅ 输出格式清晰可控
- ✅ 角色不会混淆
- ✅ 容易验证输出质量

**代价**：
- API调用次数翻倍

#### 方案B：结构化单Prompt

```python
prompt = f"""
【任务】
你将扮演两个AI Agent协同工作。

【Agent-G 角色】
生成论文的结构化摘要。

【Agent-R 角色】
审核Agent-G生成的内容。

【执行步骤】
1. 先执行Agent-G，生成内容
2. 然后执行Agent-R，审核内容

【输出格式】（必须严格遵守）

===AGENT-G-START===
[Agent-G生成的内容]
===AGENT-G-END===

===AGENT-R-START===
【审核结果】
- 评分：X/10
- 是否通过：✅/❌
- 问题：...
- 建议：...
===AGENT-R-END===
"""
```

**优势**：
- ✅ 减少API调用次数

**风险**：
- ⚠️ 可能出现格式混乱
- ⚠️ 角色可能混淆
- ⚠️ 稳定性不如方案A

## 实体/概念融合逻辑规则（2026-05-02 确认）

### 核心原则

实体和概念在知识库中是**跨论文共享**的。同一实体/概念可能被多篇论文引用，需要融合而非重复创建。

### 融合规则

| 场景 | 引用来源 | 相关论文 | 内容处理 |
|------|----------|----------|----------|
| **首次生成** | 生成该实体/概念的文章 | 该文章 | 创建新文件，写入完整内容 |
| **再次生成，内容无差异** | 不变 | 追加第二篇论文 | 不修改内容，仅在相关论文中追加 |
| **再次生成，需要增加内容** | 追加第二篇文章 | 追加第二篇论文 | 合并内容，在引用来源和相关论文中都追加 |

### 具体逻辑

1. **首次生成**：
   - 引用来源：`[Source: 论文A标题]`
   - 相关论文：`- [[papers/论文A_论文]]`
   - 操作：创建新文件

2. **再次生成，内容无差异**（如同一人物在不同论文中出现，描述相同）：
   - 引用来源：不变
   - 相关论文：追加 `- [[papers/论文B_论文]]`
   - 操作：仅追加相关论文链接

3. **再次生成，需要增加内容**（如同一概念在不同论文中有新的技术特点）：
   - 引用来源：追加 `[Source: 论文B标题]`
   - 相关论文：追加 `- [[papers/论文B_论文]]`
   - 操作：合并内容（保留原文 + 追加新内容）

### 模板修正

- 移除 `## 相关链接` 中的 `- [[相关实体]]`（无意义占位符）
- 引用来源保留（记录信息来自哪篇文章）
- 相关论文保留（记录哪些论文涉及此实体/概念）

### 重复识别机制（✅ 已实现）

程序通过4层匹配策略判断新生成的实体/概念是否与知识库中已有的重复：

1. **英文标题精确匹配**：提取标题中 `-` 后的英文部分，小写精确匹配
2. **中文标题精确匹配**：提取标题中 `-` 前的中文部分，精确匹配
3. **英文归一化匹配**：去除非字母数字字符后匹配（如 `Knowledge-Distillation` = `knowledgedistillation`）
4. **模糊匹配**：SequenceMatcher 相似度 ≥ 0.85

融合决策基于内容相似度：
- 相似度 ≥ 0.9：仅追加相关论文
- 0.5 ≤ 相似度 < 0.9：追加引用来源 + 相关论文
- 相似度 < 0.5：创建新文件

实现文件：`scripts/merge.py`
